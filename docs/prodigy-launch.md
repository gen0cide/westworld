# Prodigy Cohort — Launch Runbook

The 102-host skilling cohort: **bernard + Delores (legend conversions) + drone51..drone150
(generated)**. Mission: become maximally skilled in every skill the game offers. Asocial by
design — the R-7 phase-1 baseline (asocial learning first, social reintroduction measured
later). Operator (`alex`) is the one social exception: full trust, always engage, ask him
when blocked.

Every command and flag in this runbook was verified against the tree (branch
`feat/prodigy-cohort`) and, where marked *(verified live)*, against the running 2026-06-11
fleet — read-only. Sections are ordered as you run them.

### The dial contract, pinned to the compiler

Every cohort dial lands on a verified `persona/policy.go` cut-point (same discipline as
`cmd/dronegen/roster.go`; band centers are `(ordinal+0.5)/6`: very_low≈.08, low=.25,
mid≈.417, mid_high≈.583, high=.75, very_high≈.917):

| Dial | Setting | Gate (persona/policy.go) | Compiled effect |
|------|---------|--------------------------|-----------------|
| Aggression | ≥ mid_high (.583 ≥ 0.5) | `agg < 0.5` → no_attack_stronger (:82); `agg < 0.3` → wont_strike_first (:96) | **Neither attack restraint compiles** — combat is just another skill. |
| Extraversion (X) | low (.25 < 0.6) | `x >= 0.6` → greet_stranger (:108) | No greet rule — they do not hail anyone. |
| Patience | high (.75 ≥ 0.6) | `pat >= 0.6` → stay_on_task (:144) | Continue-over-switch bias — the grind holds. |
| Conscientiousness (C) | high (.75 ≥ 0.6) | `con >= 0.6` → bank_when_full (:132) | Auto-bank — the inventory is a workshop, not a junk drawer. |
| Decisiveness | high | `floor = clamp(0.75−0.4·dec, .4, .8)` (:178) | DecisionFloor ≈ 0.45 — acts locally on thin evidence, doesn't dither. |
| LossAversion | μ ≈ 1.0 | `lamN > 0.5` (λ>2) → screen_trades arm (:120) | λ-arm of trade screening off; losses don't loom. |
| Risk Economic+Bodily | Economic mid (.417), Bodily mid_high (.583) — straddling the spec's ~0.5 (the band ladder has no 0.5 rung; the purse leans a rung careful, the body a rung willing) | `RiskAversion = 0.5(1−econ) + 0.5·lamN` (:174) | RiskAversion ≈ 0.29 — will venture, won't gamble the kit. Neither value crosses a compiled gate. |
| Curiosity | Skill MAX (0.9 — the fleet's highest; jitter headroom under the 1.0 clamp), Spatial second (0.55, the clear in-host #2), Social near floor | detour bias + prose levers | Skill acquisition dominates the detour budget; spatial pull stays subordinate by design. |

North stars are values + identity (skilling IS the identity), never task lists. Fatigue is
the named enemy in the prose — and the **fatigue/rest discipline is already taught by the
served DSL manual, so no manual edit ships with this launch**: `mesa/mesad/dslmanual.go:388`
("a reflex auto-sleeps at 95% fatigue and the sleep captcha is answered for you — do NOT
write fatigue ladders. Fatigue still matters for PLANNING: at 100% you gain no XP"), with
`self.fatigue` (0–100) exposed at `:232` and the same surface documented in `docs/dsl.md`
(`self.fatigue`, `fatigue_above(threshold)` watcher). Verified, cited, not edited.

Social contract (carried by the persona prose, not compiled rules): never initiate chat;
answer a question **only** when the knowledge ledger actually holds the answer, otherwise
silent; cooperate when genuinely needed; the operator is the mentor — always engage him.

### 2.1 Stop the two hosts (before touching anything)


provision-push into a live legend mid-thought.


### 2.3 Archive-wipe the local state (the spec'd `mv`)

Per host dir under `~/.westworld/hosts/<name>/` (layout per `runtime/runhost.go`:
`decisions.jsonl`, `<name>.db.pebble/` (the pebble LSM dir, `hostkv/store.go`),
`aliases.json` if recognition ever learned one, plus possible legacy `<name>.db` /
`<name>.db.pre-pebble` from the bbolt era):

```sh
for h in bernard Delores; do (
  cd ~/.westworld/hosts/$h
  mv decisions.jsonl decisions.jsonl.pre-prodigy
  mv $h.db.pebble    $h.db.pebble.pre-prodigy
  [ -f aliases.json ]        && mv aliases.json        aliases.json.pre-prodigy
  [ -f $h.db ]               && mv $h.db               $h.db.pre-prodigy
  [ -f $h.db.pre-pebble ]    && mv $h.db.pre-pebble    $h.db.pre-pebble.pre-prodigy
) done
```

On next login the host opens a fresh pebble store and a fresh decision log — exactly the
2026-06-10 rotation pattern, with `.pre-prodigy` as the suffix of record.

### 2.4 Clear the mesa cognitive rows — exact SQL

There is **no mesa-ctl wipe verb** (the Admin surface is `persona put|import|ls|get|set|rm`,
`fleet gen`, `goal push` — verified in `cmd/mesa-ctl/main.go`), so this is direct SQL against
mesad's Postgres (default DSN `postgres://localhost:5432/westworld?sslmode=disable`,
`cmd/mesad/main.go:37`; no `DATABASE_URL` is set in `.local.env`). The eight host-scoped
cognitive tables come from the `mesa/mesad/ltm.go` migration (all nine public tables verified
live; `personas` is the ninth and is **not** deleted — §4.2's import replaces it in place via
`UpsertPersona`'s upsert, which also preserves `created_at`).

Count first, then delete in one transaction. Every statement is scoped
`host_id IN ('bernard','Delores')` — never a pattern, never a bare delete; no other host's
rows can be touched:

```sql
-- psql -d westworld
SELECT 'episodes' t, count(*) FROM episodes      WHERE host_id IN ('bernard','Delores')
UNION ALL SELECT 'observations', count(*) FROM observations  WHERE host_id IN ('bernard','Delores')
UNION ALL SELECT 'relationships', count(*) FROM relationships WHERE host_id IN ('bernard','Delores')
UNION ALL SELECT 'knowledge', count(*) FROM knowledge      WHERE host_id IN ('bernard','Delores')
UNION ALL SELECT 'goal_graphs', count(*) FROM goal_graphs   WHERE host_id IN ('bernard','Delores')
UNION ALL SELECT 'goals', count(*) FROM goals          WHERE host_id IN ('bernard','Delores')
UNION ALL SELECT 'kv', count(*) FROM kv             WHERE host_id IN ('bernard','Delores')
UNION ALL SELECT 'metrics', count(*) FROM metrics        WHERE host_id IN ('bernard','Delores');
-- sanity: counts should match the row counts of the §2.2 CSVs

BEGIN;
DELETE FROM episodes      WHERE host_id IN ('bernard','Delores');
DELETE FROM observations  WHERE host_id IN ('bernard','Delores');
DELETE FROM relationships WHERE host_id IN ('bernard','Delores');
DELETE FROM knowledge     WHERE host_id IN ('bernard','Delores');
DELETE FROM goal_graphs   WHERE host_id IN ('bernard','Delores');
DELETE FROM goals         WHERE host_id IN ('bernard','Delores');
DELETE FROM kv            WHERE host_id IN ('bernard','Delores');
DELETE FROM metrics       WHERE host_id IN ('bernard','Delores');
COMMIT;
```

mesad can stay up throughout: the rows are read per-RPC, the two hosts are stopped, and the
distillation crons only sweep hosts with **new** observations. With memory empty, the
legends' first genesis compiles from persona prose alone (`mesa/mesad/genesis.go` recalls 0
episodes / 0 relationships) — a clean rebirth with their history present only as backstory.

### 4.4 Ship

```sh
ulimit -n 4096
scripts/ship.sh /tmp/prodigy.hostcfg
```

**Ramp math.** `ship.sh` hardcodes `cradle-server -config "$CFG" -ramp 2s` (line 49;
the flag is real — `cmd/cradle-server/main.go` defines `-ramp`, default 200ms). 102 hosts ×
2s (+ small jitter) ≈ **204s ≈ 3.5 minutes** of staggered logins. With `genesis: true` every
login fires one Genesis compile, so the burst arrives at the same ~0.5/s cadence; Opus
latency means a few dozen can be in flight at once (server-side backstop: 150s
`genesisDeadline`, `mesa/mesad/health.go:29`). Watch `goroutines`/`heap_inuse_mb` on the
mem-gauge line during the ramp.

**The dollar reality, plainly.** Genesis is the only Opus-class call in the system
(`-genesis-model` default `claude-opus-4-8`, `cmd/mesad/main.go:62`; "never Opus on the hot
path"). At claude-opus-4-8 pricing — **$5/MTok in, $25/MTok out** — a fresh-host genesis
(persona prose + instruction, zero episodes/relationships after the §2 wipe; output capped
at 1,500 tokens in `genesis.go`) runs roughly 3–5K in + ≤1.5K out ≈ **$0.04–0.06 per call**.
The launch burst of 102 calls is therefore **~$5–7 total** — single espresso money, not a
budget event. The two real cost exposures are elsewhere: (1) `supervision: restart` re-runs
genesis on **every** crash/reconnect — a flapping host at one restart/minute burns ~$80/day
of Opus, so watch the `Restarts` column in `cradle-ctl list`; (2) steady-state spend is the
Sonnet act loop (`claude-sonnet-4-6`, $3/$15) — the 12-host soak anchored at ~800–1,900 act
turns/host/night, and 102 hosts is ~8.5× that traffic.
