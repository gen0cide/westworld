# host ↔ mesa — the gRPC wire contract (v2, game-native)

> **STATUS: BUILT** — verified against code 2026-06-10, branch HEAD `0bfa818`.
> The message/RPC SSOT is **`mesa/proto/mesa.proto`** (pure proto3, package
> `westworld.mesa.v2`, generated by buf — root `buf.yaml`/`buf.gen.yaml` — into
> `mesa/proto/*.pb.go`, no CGO). This doc is the prose companion that the proto
> header cites: what each service is FOR, where each RPC is called on the host,
> how it degrades, and the auth model. Server: `mesa/mesad` (binary
> `mesa/cmd/mesad`). Host client: `mesa/client` (`GRPCClient` is the wire
> transport; `StubClient` is the always-offline default). Open protocol work
> lives in [`docs/TODO.md`](../TODO.md) — notably M-3 (live `PERSONA_REVISION`
> push), M-6 (mTLS, `ListPersonas` paging), C-19 (genesis remainder).

The contract is distilled around the GAME LOOP. The host's currency is **DSL +
game state**: it ships game SITUATIONS up and receives game MOVES (DSL) down.
mesa's primary output is DSL — routines to run, routines to author, single
actions — NOT abstract "decisions". Each isolated host owns its OWN connection
and authenticates as its own `host_id`; the cradle daemon (`cmd/cradle-server`)
is a lifecycle manager, **not in the data path** — a cradle-spawned host and a
`cmd/host` standalone host use the identical host→mesa path.

Domain mapping (each wire message mirrors a `mesa/client` Go type 1:1):
`Move` ⇿ a conductor `Intent` (`runtime/mesa_director.go` `moveToIntent`) ·
`Decision` ⇿ `brain.Decision` (`mesa/client/adapters.go` `AsStrategist`) ·
`Episode.relation` ⇿ a `limbic` trust delta · `Provisioning.persona_json` ⇿
`persona.Persona`, which the **host** compiles the rest of the way
(`persona.CompilePolicy` → the pearl table; `runtime/runhost_bootstrap.go`
`applyPersona`).

## Services at a glance

| Service | RPCs | What it is |
|---|---|---|
| `Game` | Act · Decide · Chat · AnalysisInterpret · ExtractDialog | the LLM seams: agent step, option pick, speech, operator override, reactive extraction |
| `Knowledge` | Recall · FetchRelationships · FetchGoal · FetchKnowledge · FetchGoalGraph | recall (RAG) + the four cold-start bootstrap pulls |
| `Journal` | Remember · RecordObservations · SyncRelationships · SyncGoal · SyncKnowledge · SyncGoalGraph · ReportMetrics | the async write path: episodes, the perception firehose, AuthLocal snapshot mirrors, telemetry |
| `KV` | Put · Get · Delete | generic host-namespaced blob mirror (the `memory.Manager` remote tier) |
| `Provision` | Fetch · Subscribe · Genesis | compiled behavior down: persona pull, directive push stream, session-genesis compile |
| `Admin` | PutPersonas · GetPersona · ListPersonas · DeletePersona · PushGoal | operator control plane (`cmd/mesa-ctl`), admin-token gated |

mesad implements all six on one `Server` (`mesa/mesad/server.go`) behind three
fixed LLM tiers: `actLLM` (Sonnet-class, cached system prefix, high volume),
`decideLLM` (Haiku-class), `genesisLLM` (Opus-class, rare).

## Game — the agent interface (the core)

**Act — the agent step** (unary) — `Situation{goal, trigger, world, recent,
affect, hints}` → `Move`. Host caller: `runtime/mesa_director.go` `Next` builds
the situation (position, hp, inventory, nearby entities, dialog options, recent
transcript, epistemic hints), calls Act, and converts the Move to a conductor
`Intent`. Server: `mesa/mesad/act.go` `act` on the `actLLM` tier, persona-
grounded. `Move` is a tagged union — `MoveKind` selects the fields:

```
RUN_ROUTINE{routine_path, args}        → run a library routine
WRITE_ROUTINE{routine_name, dsl_source, args, quarantined}
                                       → host PARSES + VALIDATES + runs it under
                                         the pearl Gate; quarantined ⇒ run-once,
                                         logged, never auto-promoted to the library
DIRECT_ACTION{verb, action_args}       → one gated DSL action
IDLE{idle_seconds}                     → wait
```

Orthogonal to `kind`, the Move carries the planner's **goal-lifecycle
declaration**: `goal_op` (`""` | `"done"` | `"abandoned"` | `"adopt"`),
`goal_text` (the open-goal successor to queue on `adopt`), and `goal_progress`
(0..1, honored only while `goal_op == ""`). The host applies it
deterministically in `applyGoalOp` (`runtime/director_goals.go`) — the planner
NAMES the lifecycle state, the host records it as a durable goal-graph write.
DEGRADE: any Act error → the director idles 3s and retries next turn; the loop
never blocks on mesa.

**Decide** (unary) — `Choice{question, options, affect}` → `Decision{choice,
reasoning, confidence}`. The narrow in-routine option pick. Host path
(`runtime/actions_flow.go` `dslDecide`): `pearl.TryDecide` answers locally
on a hit (free, authoritative); on a miss the pearl may still re-bias the
option order, then the locally-keyed decision cache
(`runtime/decision_cache.go`) is consulted, and only a cache miss reaches mesa
via `AsStrategist`. Server: `decideLLM`, in-persona system prompt. The wire
carries `cache_key`/`cache_ttl_seconds` hint fields, but mesad does not set
them and the host derives its own cache key — they are reserved.
DEGRADE: error → a DSL failure (`SERVER_REJECTED`) the routine handles.

**Chat** (unary, two modes) — `ChatTurn{from, message, recent, mode, topic}` →
`ChatReply{text, speak}`. The fast, cheap speech seam (`decideLLM`), off the
Act loop. `mode == ""|"reply"`: answer a player's line — the knowledge-grounded
social reply in `runtime/runhost_bootstrap.go` (rate-limited, honesty facts
supplied by the host). `mode == "ask"`: the PROACTIVE twin (`Client.Ask`,
`mesa/client/grpc.go`) — the host has a goal-blocking unknown and an
interlocutor in range (`runtime/speech.go`), so mesa composes ONE in-character
question about `topic`; the host decided WHEN and WHO deterministically, mesa
only supplies the words. `speak == false` ⇒ stay silent; silence is also the
error contract.

**AnalysisInterpret** (unary) — `AnalysisDirective{directive, state}` →
`AnalysisVerdict{kind, dsl, text}`. The operator-override classifier (flat
affect, NOT in-persona): `kind` is `command` (the one DSL statement to run) |
`answer` (a terse literal reply) | `hypothetical`. Host caller:
`runtime/analysis.go` during ANALYSIS MODE. Server: `decideLLM`.

**ExtractDialog** (unary) — `DialogWindow{speaker, speaker_role, window,
persona_snippet, active_goal, open_questions}` → `ExtractedDialogSet{claims,
intent}`. The reactive tier (speed-2): extract `DialogClaim`s for the knowledge
ledger + classify ONE `DialogIntent`; the host decides the interrupt
deterministically from `intent.urgency` (`immediate|high` ⇒ preempt) — the
LLM's job is extraction, never control flow. Claim `provenance` is advisory:
the host overrides it from the speaker role on writeback (a player cannot claim
system authority). Host caller: `runtime/reactive.go` (8s budget). Server:
`decideLLM` by default, escalating to `actLLM` on nuance (`extractWantsNuance`)
— never Opus. DEGRADE (and `StubClient`): no claims, a low-urgency statement.

## Knowledge — recall + the cold-start bootstrap pulls

**Recall** (unary) — `Query{text, kind, top_k}` → `KnowledgeSet`. RAG over the
host's own episodes: `mesa/mesad/ltm.go` ranks by pgvector cosine similarity
when an embedder is wired, degrading to Postgres full-text + recency; empty
(not an error) when no LTM is wired. `KnowledgeItem` round-trips `entity` and
`importance` so a cold-start journal rebuild keeps its `Salient()` ordering.
An empty query returns episodes in recency order — that is how
`bootstrapJournalFromMesa` (`runtime/memory_journal.go`) reconstitutes a fresh
host's journal from nothing. Host surfaces: the DSL `recall(...)` verb and the
`AsRetriever`/`AsRemote` adapters (`mesa/client/adapters.go`).

**FetchRelationships / FetchGoal / FetchKnowledge / FetchGoalGraph** (unary,
`HostRef` →) — the four cold-start bootstrap pulls, mirror twins of the Journal
`Sync*` pushes below. A restarted host warm-starts state it never explicitly
wrote: the trust ledger (`runtime/limbic.go`), the standing objective + progress
(`runtime/memory_journal.go`; `Goal.found` distinguishes "no row"), the
consolidation-cron-distilled world-knowledge ledger (`runtime/knowledge.go` —
wire mirror of `cognition/knowledge.Entry`, Beta(α,β) beliefs + provenance),
and the insight-cron-grown intention graph (`runtime/goalgraph.go` — wire
mirror of `cognition/goalgraph`). All four are AuthLocal: the host owns the
truth; mesa is the durable mirror + bootstrap source.

## Journal — the async write path (never on the hot loop)

**Remember** (client-stream) — `Episode{idempotency_key, kind, text,
importance, relation, tags}` → `RememberAck{accepted, deduped}`. Milestone
episodes (kill/death/trade/objective/...) mirrored UP, fire-and-forget:
`mirrorEpisode` (`runtime/memory_journal.go`) pushes each captured episode on
its own goroutine with a 5s budget. The **local journal stays the source of
truth**; the content-derived idempotency key makes at-least-once resends dedup
server-side (`ON CONFLICT`). An optional `RelationDelta` carries a lossless
trust increment.

**RecordObservations** (client-stream) — `Observation{kind, subject, text,
salience, idempotency_key}` → `RememberAck`. The salience-gated perception
FIREHOSE (`runtime/observation.go`) — cron fodder for distillation, distinct
from episodes (which are recalled into the planner). Free-text capture kinds
(`npc_dialog`/`player_chat`/`server_msg`/`claim_heard`) get a text-hashed
idempotency key so distinct same-second lines don't collapse; the key scheme is
mirrored byte-identically server-side (`mesa/mesad/ltm.go`
`observationIdemKey`).

**SyncRelationships / SyncGoal / SyncKnowledge / SyncGoalGraph** (unary) →
`SyncAck`. Full-snapshot AuthLocal mirrors of the four bootstrap surfaces
above. `Relationship` carries the ABSOLUTE Beta(α,β) state plus the RAW
affinity/grievance accumulator sums — the host stays the sole squash authority.
SyncKnowledge shares its per-subject upsert with the consolidation cron and
SyncGoalGraph its per-host upsert with the insight cron (`mesa/mesad/cron.go`,
`cron_insight.go`), both last-writer-wins. Pushed periodically and on shutdown
(`runtime/limbic.go` flush loop, 2s final-flush budget; 8s steady-state).

**ReportMetrics** (unary) — `MetricsReport{metrics[], at_unix}` → `SyncAck`.
Host telemetry batches (`runtime/telemetry.go`), written as a time series so
crons can aggregate ("avg Act latency", "pearl veto rate over time").

## KV — the generic state mirror

`Put`/`Get`/`Delete` over host-namespaced opaque blobs. This is the substrate
under the `memory.Manager` remote tier (`AsRemote` in
`mesa/client/adapters.go`), NOT a host-facing surface — purposeful state uses a
semantic RPC above. It replaced the old hack of shoehorning KV writes into
`"kv"` episodes. While mesa is offline, remote writes accumulate in the
manager's local write-back journal (`memory/manager.go`, persisted in hostkv)
and drain on `Flush`.

## Provision — compiled behavior down (two paths + genesis)

**Fetch** (unary) — `HostRef` → `Provisioning{persona_json, prose, goals}`.
The connect-time pull: the authoritative persona as JSON (one schema source:
the Go `persona` structs), mesa's rendered prose card, and the current goal
stack. The host validates and applies it (`applyPersona`), running
`persona.CompilePolicy` to build its pearl table + affect baseline locally.
When mesa is linked it is authoritative; a local persona file (`loadPersona`)
is only the offline branch — a failed Fetch with mesa wired leaves the host
running without a persona (warned, not fatal).

> **The pearl table never crosses the wire.** It is func-valued — predicates
> and effects are Go closures (`pearl.Table`) — so it CANNOT be serialized.
> mesa is the source of truth for WHO a host is (the persona); the host is the
> source of truth for the in-RAM machinery it compiles from that persona. Any
> policy refresh is therefore a persona/Cornerstone push + a local recompile,
> never a table transfer.

**Subscribe** (server-stream, one per host) — `SubscribeRequest{last_applied_id}`
→ `stream Directive{id, kind, payload}`. The live push path. mesad keeps a
per-host subscriber registry (`server.go` `registerSub`; a reconnect supersedes
the stale stream) and hands out monotonic directive ids. On connect it re-sends
a standing OPERATOR goal (only `Admin.PushGoal` makes a goal sticky — the
persona baseline is not re-sent, so a richer genesis goal survives). Host side
(`runtime/runhost_bootstrap.go` `subscribeDirectives`) applies what it can
live. The `DirectiveKind` reality:

| Kind | Server pushes it? | Host applies it? |
|---|---|---|
| `GOAL_REVISION` | yes — connect-time resend + `Admin.PushGoal` fan-out | yes — `host.SetLiveGoal` (soft override the director prefers until replaced) |
| `ROUTINE_UPSERT` | no (reserved) | logged only |
| `PEARL_REFRESH` | no (reserved; payload = a persona Cornerstone to recompile FROM, per the note above) | logged only |
| `PERSONA_REVISION` | no (reserved — TODO **M-3**, now unblocked by the subscriber registry) | logged only |
| `TRUST_DECAY` | no (reserved — TODO C-24) | logged only |
| `REVERIE_REBASELINE` | no (reserved — TODO P-6) | logged only |

**Genesis** (unary) — `GenesisRequest{trigger, world_summary}` →
`GenesisResult{goal, mood, keyword_ladder, reasoning}`. The heavy
Opus-at-login session compile (`mesa/mesad/genesis.go`, `genesisLLM`): mesa
gathers the host's full history (persona + episodes + relationships + goal) and
derives the session intent, the mood baseline affect decays toward, and the
keyword→tier→action attention ladder (per-host, never hardcoded). Called once
at startup with a 90s budget (`runtime/runhost_bootstrap.go`). The designed
handler-set/policy-delta outputs never shipped — that remainder is TODO
**C-19**.

## Admin — the operator control plane

Operator-only, gated on `$ADMIN_TOKEN` (fail-closed: unset disables the whole
service; constant-time compare — `mesa/mesad/auth.go` `authenticateAdmin`).
Host tokens are never accepted here: a host must not rewrite another host's
identity. Client: `cmd/mesa-ctl` (`persona put|import|ls|get|set|rm`, `fleet`,
`goal`). Handlers reuse `Server.Register` — validate → render → persist to
Postgres → register live, no mesad restart:

- **PutPersonas** (client-stream) — bulk upsert with a per-item `BatchResult`;
  one bad persona does not fail the batch. A single put is a stream of one.
- **GetPersona / ListPersonas / DeletePersona** — registry CRUD;
  `persona_json` included on single get or `with_json` list (no paging yet —
  TODO M-6).
- **PushGoal** — `{goal, match}` sets a SOFT runtime goal override on every
  registered host matching the `host_id` glob (empty = all), marks it sticky
  for reconnects, and fans a live `GOAL_REVISION` to the currently-subscribed
  (slow subscribers are skipped, not blocked on). Returns
  `matched` (goal set) vs `pushed` (delivered live).

## Auth & transport

Every host RPC carries a per-host bearer token as gRPC per-RPC credentials
(`mesa/client/creds.go`), derived from the username by the shared leaf
`mesa/auth.HostKey` (client alias: `mesa/client/hostkey.go`). mesad's
interceptor resolves token → `host_id` and binds the AUTHENTICATED identity
into the request context (`mesa/mesad/auth.go`); handlers read
`hostIDFromContext` and never trust a `host_id` in the request body. Transport
is insecure today (trusted/local link); the production path is TLS/mTLS — flip
`RequireTransportSecurity`, and with mTLS the client cert can replace the token
(TODO M-6). The connection is lazy (`grpc.NewClient`), so `Healthy()` treats
Idle/Connecting as healthy-pending.

## The throughline

The host's behavior IS `{routine library (author-written, with mesa-authored
additions via Act)} + {selection policy (the HOST-compiled pearl table, from
the mesa-served persona)} + {escape hatch to Game.Act for novel situations}`.
**DSL is the lingua franca.** Act (synchronous "write me a script now") and
Provision/`ROUTINE_UPSERT` (async "here's your improved library" — reserved,
not yet pushed) are the same currency at two timescales. ALL mesa-authored DSL
runs through the local parse/validate + pearl Gate; inline-authored routines
are additionally **quarantined** (run-once, logged, no auto-promotion).

## Semantics

| Capability | RPC | Style | Host budget (verified) | On failure |
|---|---|---|---|---|
| Act | Game.Act | unary | run ctx (no per-call deadline) | idle 3s, retry next turn |
| Decide | Game.Decide | unary | run ctx | DSL failure; pearl bias / cache absorb repeats |
| Chat / Ask | Game.Chat | unary | run ctx (rate-limited callers) | stay silent |
| ExtractDialog | Game.ExtractDialog | unary | 8s (`runtime/reactive.go`) | no claims, low urgency |
| Recall | Knowledge.Recall | unary | 8s (bootstrap path) | empty set, proceed |
| Remember | Journal.Remember | client-stream | 5s, fire-and-forget | drop; local journal authoritative |
| Observations | Journal.RecordObservations | client-stream | 5s, fire-and-forget, bounded in-flight cap | drop; firehose is lossy by design |
| Sync\* / Fetch\* | Journal / Knowledge | unary | 8s (2s final flush) | retry next flush / skip bootstrap |
| Provision | Provision.Fetch | unary | 10s | warn; host runs without a persona |
| Genesis | Provision.Genesis | unary | 90s | persona north-star goal |
| Subscribe | Provision.Subscribe | server-stream (1/host) | connection lifetime | reconnect; operator goal re-sent |

Universal degrade rule: `mesaclient.ErrOffline` (and any RPC error) means
"fall back to local behavior", never a hard failure — a host runs fully
self-contained on `StubClient`.

## History

The contract shipped in two phases, as designed: first as plain Go structs
behind the `mesaclient.Client` interface (`StubClient` default), then the
tracked proto + buf-generated stubs with `GRPCClient` swapped in behind the
same interface — the host seams were unchanged by the swap. The
`westworld.mesa.v2` package name is the surviving artifact of the v1→v2
(game-native) redesign.
